# 异常伤害过滤白话版

## 一、要解决什么问题

游戏里玩家正常打怪一下大概打 10 ~ 600 点。但有时候地图脚本会"作弊"打出 1000 点假伤害，把伤害榜搞乱。

**目标**：自动识别并屏蔽这种 1000 点假伤害，但不能误伤正常的 600 点真伤害。

## 二、想象成"三道安检门"

我们给每个玩家设三道安检门，每打一下都要**全部通过**才算异常（被拦）。任何一道放行就算合法。

```
        进来一刀 X 点伤害
              ↓
    ┌─────────────────────┐
    │ 门 1：金额够大吗？      │ 看 X 是不是大于一个固定值（800）
    │ 简称"地板门"           │
    └─────────────────────┘
              ↓ 是
    ┌─────────────────────┐
    │ 门 2：比平时单刀大很多？  │ 看 X 是不是大于"最近 100 刀的高位水平 ×3"
    │ 简称"P95 门"           │
    └─────────────────────┘
              ↓ 是
    ┌─────────────────────┐
    │ 门 3：比当前节奏猛很多？  │ 看 X 是不是大于"最近几秒平均出手 ×1.5"
    │ 简称"DPS 门"           │
    └─────────────────────┘
              ↓ 三门全过 = 异常
              拦截！
```

## 三、三道门各自管什么

### 门 1：地板门（绝对值 800）

**最简单的一道**：只看伤害数字大不大。

- 一刀 ≤ 800 → **直接放行**，后两道门都不用查
- 一刀 > 800 → 进入门 2

**为什么是 800？** 因为合法最大是 C 武器 600，异常是 1000。**800 正好夹在中间**——比 600 大留 33% 余量，比 1000 小给拦截留余地。

> 这道门是新加的，作用是"画一条所有合法伤害都不会超过的红线"，先排除掉 99% 的正常情况。

### 门 2：P95 门（比日常水平大很多）

**记小本子**：给每个玩家记一本"最近 100 刀的小本子"，每打一刀就把数字记进去（满 100 后挤掉最老的一个）。

**P95 = 第 95 大的数字**——简单说就是"日常水平的高位线"。

- 玩家主用 A 武器（10/刀）：本子里基本都是 10 → P95 = 10
- 玩家主用 B 武器（300/刀）持续 100 秒+：本子里都是 300 → P95 = 300
- 不停切换三武器：本子被高频的 A 占据 → P95 ≈ 10

**门 2 阈值 = P95 × 3**：日常的 3 倍。

- 一刀 > P95 × 3 → 进入门 3
- 一刀 ≤ P95 × 3 → **放行**（属于"日常波动 + 暴击"范围）

> 这道门解决"自适应"问题——玩家是新手或老手，门 2 的阈值会根据本人水平自动调整。

### 门 3：DPS 门（比当前节奏猛很多）

**看最近 5 秒**：玩家最近 5 秒一共打了多少？除以"有打过的秒数"，得到**当前出手节奏**（DPS_active）。

- 玩家持续每秒打 200 → DPS_active = 200
- 玩家间歇打（打 1 秒停 1 秒）→ DPS_active 仍按出手秒算 = 真实节奏，不被空闲秒稀释
- 玩家停打 5 秒以上 → DPS_active = 0

**门 3 阈值 = DPS_active × 1.5**：当前节奏的 1.5 倍。

- 一刀 > DPS_active × 1.5 → 三门全过 → **拦截！**
- 一刀 ≤ DPS_active × 1.5 → 放行

> 这道门解决"连击保护"——玩家持续高强度输出时阈值自然变宽，合法暴击/连击不会被误伤。
>
> **特殊情况**：玩家停打 5 秒后，DPS_active = 0，门 3 阈值 = 0，任何伤害都自动算"超过节奏"——这种时候只看门 1 + 门 2 决定。

## 四、参数总览

| 参数 | 值 | 含义 | 大白话 |
|---|---|---|---|
| `outlierAbsoluteFloor` | **800** | 绝对地板 | 不到 800 不查 |
| `p95OutlierMultiplier` | **K_p = 3** | P95 倍数 | 比日常单刀大 3 倍才算 |
| `dpsActiveMultiplier` | **M_d = 1.5** | DPS 倍数 | 比当前节奏快 1.5 倍才算 |
| `p95WindowSize` | 100 | 小本子大小 | 记最近 100 刀 |
| `p95MinSamples` | 20 | 最少样本 | 没打够 20 刀不启用门 2/3，靠门 1 自己撑 |
| `bufferDelayTicks` | 2 | 延迟检查 | 伤害先入队，等 2 tick 再判断（等死亡信号到位）|

## 五、配套规则（防作弊）

### 规则 1：被拦的伤害不入小本子

如果某一刀被三门拦下（异常），就**不写进小本子，也不进 5 秒桶**。否则下次同样大的异常，因为小本子被拉高了反而拦不住——这叫"自污染"。

### 规则 2：致死的机制刀仍归属玩家

地图脚本"机制斩杀"打 100K 致死时：
- 伤害账户**不加** 100K（避免伤害榜虚高）
- 但击杀**仍归属玩家**（避免人头丢失）

通过一个叫 `wasLethalAtBuffer` 的小标记区分：
- 入队时 victim 已经是致死血量 → 标记为 `lethal-mechanism`（仍写归属）
- victim 没死 → 标记为 `outlier`（只过滤）

### 规则 3：致死单刀不入小本子

玩家高光击杀（单刀打死 BOSS）的"高光数字"也不入小本子。这种数字是偶然的，不该污染日常分布。

## 七、L 面板（实时监控）

按一下 L 键，HUD 顶部显示这些数字让你观察：

```
══════════════════════════════════════════════════════════
 ◆ FILTER · P95=10(100/100)×3=30  DPS=200×1.5=300  floor=800
══════════════════════════════════════════════════════════
```

含义：
- `P95=10`：你最近 100 刀的高位水平是 10
- `(100/100)`：本子已记满 100 刀
- `×3=30`：门 2 阈值
- `DPS=200×1.5=300`：门 3 阈值
- `floor=800`：门 1 阈值（固定）

按 L 进入子页签还能看到：
- 三道门各自拦下的次数（today's catch）
- 每个队友的 P95 / DPS / 阈值
- buffer 队列长度

## 八、有什么能调的（实战调参）

| 现象 | 调什么 | 怎么调 |
|---|---|---|
| 异常 1000 偶尔漏过 | `outlierAbsoluteFloor` | 800 → 700（更激进，但小心合法 700+ 单刀） |
| 合法 600 偶尔被拦 | `outlierAbsoluteFloor` | 800 → 900（更保守） |
| C 武器爆发后异常漏过 | `M_d` | 1.5 → 1.2 |
| 玩家长期主用 B 后误判 | `K_p` | 3 → 2 |
| 想关闭整个过滤器 | `useDamageBuffer` | true → false |

## 九、为什么这个方案好

1. **三道门互相补充**：单一指标都有死角，三个 AND 起来漏洞极少
2. **地板门是底线**：保证合法 600 伤害**永远不会**被拦
3. **P95 门自适应**：玩家水平不同自动调整，不需要服务器手动配置
4. **DPS 门防误伤**：合法连击/暴击因节奏跟得上不会被拦
5. **被拦的不入样**：避免异常自我强化的恶性循环
6. **致死特殊路径**：机制刀致死时击杀归属仍正确，伤害榜不虚高

## 十、为什么不用黑名单？

黑名单（"看到 1000 就拦"）有两个问题：
1. **写死**：异常值变成 1001 就漏了
2. **副作用**：如果将来有合法 1000 真伤就误伤

而本方案是**统计学方案**——不管异常值具体是多少，只要"明显超出该玩家日常水平 + 超出绝对地板"，都能识别。**对未知形态的异常也有效。**

## 一句话总结

**三道门 AND**：金额够大（>800）+ 比日常单刀大很多（>P95×3）+ 比当前节奏猛很多（>DPS×1.5）—— **三个都满足才算异常**。地板门 800 是关键创新，它在合法 600 和异常 1000 之间画了一条数学上明确的红线。
