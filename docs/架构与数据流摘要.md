
## 一、HUD 层（`hud`）

### `HealthBarRenderer.java`
- **作用**：在屏幕左下/右下绘制**个人**生命条、法力/鲜血条、心形/金币等装饰；右侧列队友条（可横向/纵向布局），v6.6 起在条下方叠 **关:/局: 两行嵌入式统计**（依赖 `TeammateStatsLine` + `currentStatsData` 等）。  
- **关键 API**：`static void render(DrawContext, RenderTickCounter, HealthData, StatsData)`；大量私有 `render*` 与尺寸常量。  
- **模式**：只消费 `HealthData`/`StatsData` 与 `ModConfig`；不直接走网络，统计数字由上层注入的 `StatsData` 与队友行用的 `ClientStatsCache` 装配链间接覆盖。

### `StatsRenderer.java`
- **作用**：在 HUD 上绘制 **ViewStats/属性** 多行 `Text`（`StatsData.getLines()`），受 `ModConfig.statsVisibility`、`statsX/Y`、`statsColumns` 控制。  
- **关键 API**：`static void render(DrawContext, StatsData)`。  
- **模式**：无网络；纯展示 + 无数据时提示语。

### `StatsTableScreen.java` + `StatsTableData.java`
- **作用**：**K 键**全屏统计表——Tab「总表 / 分关表」、列头排序、滚轮、头像与配色；数据由 `StatsTableData` 从三家 stats **快照**合并成行（`PlayerRow` / `StageBlock` / `Total`）。  
- **关键 API**：`StatsTableData.buildTotal(sort, asc)`、`buildStage()`；`StatsTableScreen` 为 `Screen` 子类，内含 Tab/列定义常量。  
- **模式**：**全部经 `ClientStatsCache`** 取数（注释写明集成服直读 server 与专用服 S2C 缓存二选一），与 `PlayerDamageStats` / `PlayerKillStats` / `PlayerTakenStats` 的 `Snapshot` 结构对齐；依赖 `StageKey` 与 `StageNameRegistry`（分关名）。

### `DamagePanelRenderer.java` + `DamagePanelScreen.java`
- **作用**：**L 键**或 HUD 上绘制「伤害分配」开发向面板；无大背景、可 **[X] 清零 / 详略 / Session·Current 切片**（`currentScopedSnapshot()` 结合 `ModConfig`）；`DamagePanelScreen` 负责拖拽、关闭、将绘制委托给 `drawCore`。  
- **关键 API**：`drawHud` / `drawCore` / `currentScopedSnapshot()` / `measureHeight` / `handleButton` / `hitTestButton`；快照类型为 `PlayerDamageStats.Snapshot`（集成服含未归属等；专用服经 cache 可能缺省段为 0）。  
- **模式**：**集成服**可视为直接读 `PlayerDamageStats`；**远程**通过同一套 `Snapshot` 形状经接入层提供（与 `ClientStatsCache` 设计一致，见下）。

### `TeammateWorldRenderer.java`
- **作用**：在 **3D 世界**实体头顶画队友血条（及 `renderMobHealthBar` 等），由 `WorldRendererMixin` 在 `renderEntity` 末尾调用。  
- **关键 API**：`renderHealthBar(...)`；读 `HealthData.getTeammateMap()` 按名匹配。  
- **模式**：与屏幕 HUD 的 `HealthData` 同步；受 `ModConfig.showTeammateHeadHP` 等开关控制。

### `TeammateStatsLine.java`
- **作用**：为队友 HUD **装配**「关: / 局:」行数据：`ofSession` / `ofStage`，从 **`ClientStatsCache`** 取 dealt/taken/kills/assists 及关内 DPS（`recent5sDps`）。  
- **关键 API**：`Line` record；`ofSession(UUID)`、`ofStage(UUID)`；`compact(long)` 数字缩写。  
- **模式**：**彻底与 server 包解耦**，仅经 `ClientStatsCache` + `StageKey`。

### `StageNameRegistry.java` / `StageLocation.java` / `ClientStageLocation.java`
- **`StageNameRegistry`**：从资源 `stage_name_map.json` 加载 **关卡 id → 中英名**，`localizedName(Kind, id, ...)` 供 HUD/聊天等用；**不**依赖地图资源包。  
- **`StageLocation`**：定义 `Kind` / `GameOverPhase` / `Snapshot`（**ordinal 须与服务端 `StageProbeServer` 对齐**）、`Snapshot.fromPayload(StagePayload)`、`formatted()` 富文本；`probe()` **仅** `return ClientStageLocation.current()`。  
- **`ClientStageLocation`**：存 ** volatile** 的最近一次 `StagePayload` 解包结果；网络线程写、渲染线程读。

## 二、Mixins（`mixin` / `server/mixin`）

### `BossBarHudAccessor.java`
- **作用**：`BossBarHud` 的 **Accessor**，暴露 `Map<UUID, ClientBossBar> getBossBars()` 供他处需要时访问（本 mod 的 `BossBarHudRenderMixin` 主要用 Shadow）。

### `BossBarHudRenderMixin.java`
- **作用**：在 `BossBarHud.render` **HEAD/RETURN** 间临时从 `bossBars` 移除 `HealthData.getHiddenBarUUIDs()` 中的条，**避免**与自定义 HUD 叠画；render 后还原。

### `WorldRendererMixin.java`
- **作用**：`renderEntity` **TAIL** 调用 `TeammateWorldRenderer.renderHealthBar` 与 `renderMobHealthBar`，把 3D 条hook进实体渲染管线。

### `TeammateHealthMixin.java`（`EntityRenderer`）
- **作用**：`renderLabelIfPresent` 开头：在 **mob 头顶模式** 下可 **cancel** 原标签，改由 CTT 画；在 **队友模式** 下识别「数字+心」类 vanilla 血条并 cancel，避免与自定义重复；含 `findTeammate` / `findMobWithHealth` 等。

### `VanillaHealthHiderMixin.java`（`PlayerEntityRenderer`）
- **作用**：专对 **玩家** 标签：匹配「纯数字开头」的 vanilla 生命显示时 **cancel**（与队友头顶 HP 显示策略配合）。

### `server/mixin/ScoreboardUpdateMixin.java`
- **作用**：**服务端** `Scoreboard.updateScore` **唯一入口**，按 objective 名/criterion 分派：  
  - `RedHearts` → `DamageProbe.recordFromRedHearts`（可配置主数据源）  
  - `DamageShower` → `DamageProbe.record`  
  - `*DMG` → `AttackerProbe.record`（`ScoreDeltaTracker` 用增量）  
  - `damage_dealt` / `carrot_on_a_stick` 等 → `PlayerHitLog` / `PlayerFireLog`  
- **模式**：`ServerScoreboard` 过滤；是 **计分板伤害进入本 mod 的主闸门**（与已读的 `AttackerProbe` / `CttStatsServer` 形成链）。

## 三、网络（`network`）

### `StagePayload.java`
- **S2C** 自定义包：`kind/tier/floor/stageNum/breakRoom/miniGame/gameOverPhase/checkpoint`；`PacketCodec` 编解码；id `ctt-health-display:stage_location`。

### `StatsSnapshotPayload.java`
- **S2C 全量 stats 快照**（v1/v2 版本字段）；含 session 元数据、**阶段列表** `StageEntry[]`、**玩家行** `PlayerEntry[]`（含 per-stage 行、v2 `recent5sSum`）；`StageEntry.toKey()` 还原 `StageKey`。

## 四、服务端统计与持久化（`server`）

### `PlayerKillStats.java`
- **作用**：**击杀/助攻/boss 击杀/未归属击杀** 等，与 `AttackerProbe` 归属层、致死逻辑挂钩；`syncStart/clear/stop` 与 `PlayerDamageStats` 联动。  
- **关键 API**：`Entry`、`Snapshot`、`toNbt`/`fromNbt` 等；静态 `ConcurrentHashMap` + 原子类型。

### `PlayerTakenStats.java`
- **作用**：**承伤** 累计（`PlayerTakenProbe` 与 `DamageTook` 管线）；含 **按 `StageKey` 分桶**；与 session 控制同步于 `PlayerDamageStats`。  
- **关键 API**：`addTaken` 等（在全文其他处调用）；`Snapshot`、`stageBuckets`。

### `PlayerDpsTracker.java`
- **作用**：每玩家 **5 个 1 秒桶** 滑窗，`PlayerDamageStats.add` 后 `onDealt`；`recent5sSum` 供 HUD 关行 DPS 与 `StatsSnapshotPayload` v2。

### `StatsPersistenceManager.java`
- **作用**：世界目录 `data/ctt_stats.dat`（gzip NBT）**持久化三家 stats + `StageBoundaryDispatcher` 时间表现** + **history 归档**（GameID 跳变时由 dispatcher 钩子等触发）。  
- **模式**：`load`/`saveNow`/`onTickEnd` 节流；`archiveAndReset` 与 `StageBoundaryDispatcher.onSessionChange` 联动。

### `StatsSnapshotBroadcaster.java`
- **作用**：`END_SERVER_TICK` **约 1 秒** 或 **玩家 JOIN** 时 `build()` + `ServerPlayNetworking.send` 广播 `StatsSnapshotPayload`；数据源为三家 `snapshot()` + `StageBoundaryDispatcher` 阶段元数据 + `PlayerDpsTracker`。

### `StageReportBroadcaster.java`
- **作用**：`StageBoundaryDispatcher` 在**退关/切关**时调 `onStageExit(StageKey)`，向**全服在线**发**聊天栏多行战报**（本关 + 全局 session），按 `viewerUuid` 高亮自己；`StageNameRegistry` 取中文名、同 stageKey 短时间 **去重**。

### `StageBoundaryDispatcher.java`
- **作用**：服务端 **权威** 每玩家 `StageKey`、是否 **`isCollecting`**（非战斗关不写 stats）、`lastSeenStageKey`、阶段进入/离开时间、GameID 与 **enter/exit/session 监听器**；`updateFromPayload` 与 `StageProbeServer` 同步。  
- **被依赖**：三家 stats 的 `add`、持久化、战报、快照里的阶段列表。

### `StageProbeServer.java`
- **作用**：每 tick 读 **scoreboard holder + 玩家 `CTT` tag** 算 `StagePayload`，diff/心跳 后 `ServerPlayNetworking.send`，并调 `StageBoundaryDispatcher.updateFromPayload`；**JOIN/断开** 清缓存。

### `StageKey.java`
- **作用**：五元组 `(gameId, tier, floor, stageType, stageNum)` + `NULL` session 占位；`toNbt`/`fromNbt`；`equals/hashCode` 作 Map key。

### `WeaponDamageRegistry.java`
- **作用**：`weapon_damage_seed.json` + **vanilla 弓弩三叉戟** 硬编码表；`getInfo` / `canProduce` / `weaponsOfType`，供 `AttackerProbe` 做武器-伤害类型 **守卫**。

### `PlayerInventoryIndex.java`
- **作用**：每 **5 tick** 扫在线玩家，缓存 **主手 custom_data 键** 与 **主手+快捷栏+背包** 键集（召唤物用），`Snapshot` 供 `AttackerProbe` 事件内 O(1) 风格查询，避免每击全包扫描。

## 五、客户端缓存（`client`）

### `ClientStatsCache.java`
- **作用**：**统一读入口**：`CttStatsServer.getServer() != null` 时 **委托** 原 `Player*Stats` / `StageBoundaryDispatcher` / `PlayerDpsTracker`；否则读 **`StatsSnapshotPayload latest`**，构造与 server 同形的 `Snapshot` 与按 UUID/stage 的 getter。  
- **关键 API**：`update(StatsSnapshotPayload)`、`reset()`、`damageSnapshot()`/`killSnapshot()`/`takenSnapshot()`、`getDealt`/`getTakenAt`/`lastSeenStageKey`/`recent5sDps` 等。  
- **这是 HUD/K 表/L 面板的「专用服数据路径」核心**。

## 六、配置界面（`config`）

### `ConfigScreen.java`
- **作用**：**客户端** Mod 菜单主屏：队友条/属性块 **拖拽预览**、刷新间隔、头顶 HP、mob HP 模式、柱宽、`stats` 显隐与列数、各条显隐等；改 `ModConfig` 并保存。

### `ServerConfigScreen.java`
- **作用**：**二级屏**，编辑 `ServerConfig`（聊天广播三开关、承伤广播阈值、`useRedHeartsTally`、各类过滤与阈值）；注明专用服远程改本地 JSON **不一定影响服务端**；数组型字段只读展示。

## 七、你关心的四条数据流（交叉依赖简图）

1. **伤害事件进入服务器**  
   地图/数据包对 scoreboard 的写入 → `ScoreboardUpdateMixin` → `DamageProbe`（最终伤害/红血 tally）与 `AttackerProbe`（*DMG 归属诊断）+ `PlayerHitLog`/`PlayerFireLog` 等。非 mixin 的 tick 里还有如 **承伤**（`PlayerTakenProbe`）等与 `ServerConfig` 过滤配合（你已读 `AttackerProbe` 则栈在此之上）。

2. **归因栈**  
   `AttackerProbe` 使用 **`WeaponDamageRegistry` + `PlayerInventoryIndex` 快照** 做多层 L1~L9 判定；击杀进入 `PlayerKillStats`；造伤/未归属进 `PlayerDamageStats`；与 **`StageBoundaryDispatcher.isCollecting` + 当前 `StageKey`** 决定 session 与分桶。  
   `StageProbeServer` / `StageBoundaryDispatcher` 不直接算伤害，但**决定** stats 是否写入、写入哪个 stage。

3. **统计如何到客户端 HUD**  
   - **关卡/位置文案**：`StageProbeServer` → S2C **`StagePayload`** → `ClientStageLocation.onPayload` → `StageLocation.probe()`，供 `DamagePanelRenderer` 等处显示关卡行。  
   - **局/关/表格式统计**：`StatsSnapshotBroadcaster` → S2C **`StatsSnapshotPayload`**（约 1Hz + JOIN）→ `ClientStatsCache.update` → **`HealthBarRenderer`（经 `TeammateStatsLine`）**、**`StatsTableData`**、以及 **`DamagePanelRenderer.currentScopedSnapshot()`** 等**统一**走 `ClientStatsCache`（集成服则同 JVM 直读 server 静态类）。  
   - **`StatsData`**（属性行）与 **计分板/命令** 的路径在你已读文件中，不重复；与「三家 stats 快照」是并行 UI 线。

4. **持久化流**  
   `StatsPersistenceManager` 在 **启动 load**、**tick 节流**、**退关/Session 切换**、**关服** 等写 `ctt_stats.dat`；`StageBoundaryDispatcher` 的时间表与三家 **toNbt** 同文件 schema；GameID 跳变时 **archive + reset** 进 history 上限 20 条。

## 八、与「剩余文件」直接相关的**强依赖**速查

| 从 | 到 |
|----|-----|
| `TeammateStatsLine` / `StatsTableData` / `DamagePanelRenderer` | `ClientStatsCache`（及间接 `StatsSnapshotPayload`） |
| `StageLocation.probe` | `ClientStageLocation` ← `StagePayload` |
| `StatsSnapshotBroadcaster` | `PlayerDamageStats` + `PlayerKillStats` + `PlayerTakenStats` + `StageBoundaryDispatcher` + `PlayerDpsTracker` |
| `StageReportBroadcaster` | `StageBoundaryDispatcher` 退出点、`Player*` 快照、服务端调 `StageNameRegistry`（中文） |
| `TeammateWorldRenderer` | `WorldRendererMixin`、`HealthData` |
| 隐藏原 boss 条 / vanilla 血字 | `BossBarHudRenderMixin`、`TeammateHealthMixin`、`VanillaHealthHiderMixin`、`HealthData.getHiddenBarUUIDs` |
| 计分板入口 | `ScoreboardUpdateMixin` → `DamageProbe` / `AttackerProbe` / logs |

若你需要把以上内容黏进《架构/评审》文档，可按 **包：职责 → 公开 API → 数据进/出** 三节扩写；**跨进程（专用服）** 与 **集成服** 分支以 `ClientStatsCache.isIntegrated()` 为界即可。
