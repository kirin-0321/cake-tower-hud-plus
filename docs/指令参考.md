本模组（ctt-health-display）命令（无需权限，任何玩家可用）
**per-player 订阅模式**：开关只影响你自己，不会刷屏其他玩家。下线 / 重启后自动清空订阅。

#查询自己的订阅状态 + 全局兜底状态
/ctthd broadcast status

#单独订阅/退订 伤害广播（只有你自己看到）
/ctthd broadcast damage on
/ctthd broadcast damage off

#单独订阅/退订 击杀广播（只有你自己看到）
/ctthd broadcast kill on
/ctthd broadcast kill off

#单独订阅/退订 承伤广播（只有你自己看到）
/ctthd broadcast taken on
/ctthd broadcast taken off

#单独订阅/退订 每关战绩广播（v8.x 默认关；切关时不再自动刷屏，订阅后才会看到自己那份战报）
/ctthd broadcast stage_report on
/ctthd broadcast stage_report off

#一键订阅/退订 全部 4 档（只有你自己看到）
/ctthd broadcast all on
/ctthd broadcast all off

#设置承伤广播阈值（全局，本 tick DamageTook < 阈值时不广播；默认 0=全部显示，会写盘）
/ctthd broadcast taken_threshold 100

#设置伤害广播阈值（全局，单次 damage < 阈值时不广播；默认 0=全部显示，会写盘）
#例如设 100 后只显示扣血 ≥ 100 的伤害事件，便于观察大额伤害
/ctthd broadcast damage_threshold 100

#关掉阈值过滤，恢复全部显示
/ctthd broadcast damage_threshold 0
/ctthd broadcast taken_threshold 0

说明：
- 命令仅玩家可用（控制台执行会报错）。
- 订阅状态不持久化，下线 / 重启自动清空 —— 调试性质本就是临时。
- 想让**全服都看到**广播：编辑 config/ctt-health-display-server.json 把 broadcastXxxInChat 改为 true 后重启，那是运维兜底，不是命令路径。
- v8.x 起 **每关战绩广播默认关闭**（照顾新玩家不被切关后聊天栏的大段战报刷屏）：想看自己那份战报，单机直接在游戏配置界面（ModMenu → Cake Tower HUD Plus → 服务器配置 → 聊天栏广播·每关战绩）打开，多人服务器输入 `/ctthd broadcast stage_report on` 订阅即可。

#召唤物品指令
第一步输入：/function cake_team_tower:misc/spawn_item
第二部输入：/scoreboard players set @e[type=minecraft:armor_stand,limit=1,sort=nearest] [物品类型] [数字ID]
输入完成后就会获得对应物品

物品类型有：RandomSword1这个代表垃圾和普通品质的近战武器，其中1代表垃圾和普通，如果是2则是稀有，3是罕见，4是神话，5是传说
RandomMagic1这个代表的是魔法武器，其中你会发现前面的Random不变，后面发生了改变
后面为物品类型，前面的词代表物品池，如果你想要主动充能物品则将Random后面改为Active

其中天使和恶魔交易物品特殊，还是先走第一步，但是第二步指令改为：
/scoreboard players set @e[type=minecraft:armor_stand,limit=1,sort=nearest] [天使/恶魔物品类型] [数字ID]
其中[天使/恶魔物品类型]内容为RandomASword，你会发现Random依旧保留，Sword后面没有数字，但是前面多了个A，这个A表示为天使交易，而恶魔交易为D，这个词的意思是，从天使交易物品池调出一把近战物品

[数字ID]意思为：就是很正常的你随便输入一个数字，比如输入/scoreboard players set @e[type=minecraft:armor_stand,limit=1,sort=nearest] RandomSword1 11，就会获得诅咒之刃，这是因为诅咒之刃是位于这个物品池中的11号位置，如果你输入数字后获得了盔甲架或者什么也没有，表示这个数字没有任何对应的物品
