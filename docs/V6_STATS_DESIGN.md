## 1. 整体定位

**v5 定位**：纯客户端 HUD，只读地图发来的 Boss 栏 / `/trigger ViewStats` 回复，不参与任何状态管理。

**v6 定位**：
- **单 jar，双职责**：
  - 客户端侧 = 继续承担 v5 的 HUD（零行为变化）+ 新增统计面板 UI
  - 服务端侧 = 新增采集链路，记录伤害 / 击杀 / 承伤 / 治疗，落地到存档目录
- **本地存档 = 主要测试舞台**：Fabric integrated server，客户端与服务端同 JVM；未来需要专用服务器时同一个 jar 直接丢进去即可
- **数据流**：服务端采集 → 存档持久化 → 通过自定义 payload 同步到客户端 → 客户端渲染

## 2. 四项指标定义

| 指标 | 字段名 | 权威定义 | 数据来源 |
|---|---|---|---|
| **造伤** | `dmg_dealt` | **最终对怪物造成的血量减少量**（四层血 `BlueHearts+BlackHearts+SoulHearts+RedHearts` 的累计递减值） | 监听地图 `Damage` scoreboard 写入 + 四层血 delta 校验，**与地图护甲管线末端一致** |
| **承伤** | `dmg_taken` | **玩家自身最终掉血量**（同样是四层血总下降量） | 同上，仅筛选 target 为带 `CTT` tag 的玩家 |
| **击杀** | `kills_mob` | `RedHearts` 从 `>0` 跌到 `≤0` 的瞬间（带 `E` tag 的敌对实体） | 每 tick 扫描对比 | 
| **Boss 击杀** | `kills_boss` | 同上，但 victim 带 `Boss` tag 的子集（`PlayerKillStats.bossKills`） | 击杀派发时分流 |
| **助攻** | `assists` | **同一 victim 上有"已分类"伤害贡献但不是致命一击的玩家**；**不含 killer 自身**（同一玩家在某次击杀中只占 ☠ 或 🤝 二者之一） | `recordKillAttribution()` 派发时登记 |
| **治疗**（v3 才做） | `heals_teammate` | **我治疗队友的量**（不含自我回血、不含地图自动回血） | Mixin 玩家主动治疗技能 + 目标玩家正 delta 匹配 |

**重要铁律**：
- "造伤"**一律按最终落到敌方血量上的数字**记账；元素加护、真盔甲、难度系数、Buff 乘数全部已经在地图自己的管线里结算好了，mod 只做"记录结果"
- **数值永远非负**；下限 1（地图本身 `Damage ≥ 1` clamp），我们不做二次下限
- **归属**：优先配对到玩家本人的攻击动作（5 tick 滑窗内 UUID 匹配）；追不到的记入 `unattributed`，供复盘用、不计玩家个人
- **反射伤害 / 环境伤害**：统一落到 `unattributed`
- **友军伤害**：默认记入 `dmg_dealt`（v1 不特殊处理，因为地图本身允许一定的友军误伤场景）

## 3. 统计范围与生命周期

### 3.1 Stage（单关）边界

| 事件 | 说明 |
|---|---|
| **Stage 开始** | 玩家从休息室选关进入战斗关卡（6 个 stage holder 其一由 0 跳到非 0） |
| **Stage 结束** | 玩家回到休息室（6 个 stage holder 全回到 0） |
| **Stage 粒度** | `Floor / Boss / MBoss / Dungeon / Shop / Ally / Misc` **七种都算独立 Stage** |
| **Stage 唯一键** | `(GameID, Tier, Floor, StageType, StageNum)` 五元组 |
| **在休息室期间** | 暂停统计；面板显示**上一关**数据；攻击测试假人不计入（因为休息室内不采集） |
| **检查点 (`#CheckPoint=1`)** | 不清除当前 Stage 数据（继续累计），与普通休息室返回区别对待 |

### 3.2 Session（本局）边界

| 事件 | 说明 |
|---|---|
| **Session 开始** | 服务端 `#CTT GameID` 发生跳变 |
| **Session 结束** | 玩家**开启下一把新游戏**（GameID 再次跳变） |
| **跨游戏行为** | `GameOver=100`（最终失败）或通关不清数据，数据留存到下一次 GameID 变化 |
| **Session 持久化** | 落盘到 `world/data/ctt_stats.dat`，退出存档重进继续累计 |

### 3.3 大厅 / 小游戏过滤

- `isInLobby()` = 无 CTT tag 或 `#LobbyMiniGame > 0`：**不采集**
- v1 只采集 `#LobbyMiniGame == 0`（主剧情模式）的数据
- 休息室假人：依靠"休息室期间不采集"规则天然过滤，**不做白/黑名单**

## 4. 数据聚合维度

每条采集到的事件（一次 tick 的一次伤害/击杀）会同时落入三种"桶"：

| 桶 | 含义 | 用途 |
|---|---|---|
| **player × stage** | 单个玩家在单个关卡的表现 | 分关表的每一行 |
| **player × session** | 单个玩家整局的累计 | 嵌入 HUD 下半行 + 总表 |
| **team × stage** | 整队在单关的总量 | 对照"关卡难度"与"我贡献了多少比例" |

**玩家身份**用 UUID 作为主键。改名、离线重连、同一游戏会话内数据都正确合并。

## 5. 外观设计 A：嵌入式 HUD（队友面板扩展）

### 5.1 设计目标

战斗中**不需要任何按键**就能持续看到"我自己 + 每个队友 这一关和整局各打了多少伤害"。与 v5 现有的队友血量面板无缝合并，不增加屏幕占用。

### 5.2 现状对照

**v5.3.3 当前**（每个队友一行）：
```
[头像]  玩家名字     123/200 ♥   × 2 命
```

**v6.1.x 目标**（每个队友两行；v6.6 起 ☠ 加助攻紧凑显示 `kills/assists`）：
```
[头像]  玩家名字     123/200 ♥   × 2 命
        ⚔ 1,234   ⛨ 456   ☠ 7/3          ← 当局 stage（☠ 击杀/助攻）
        ⚔ 12,340  ⛨ 4,560  ☠ 42/18        ← 整局 session
```

**另一种版面**（用户已选的"两行方案"，无 stage 数字）：
```
┌─────────────────────────────────────────────┐
│ [头] 玩家A       180/200 ♥   ×2 命         │
│      关: ⚔1234  ⛨456  ☠7/3                │
│      局: ⚔12340 ⛨4560 ☠42/18              │
├─────────────────────────────────────────────┤
│ [头] 玩家B (自己) 150/200 ♥   ×3 命        │
│      关: ⚔890   ⛨234  ☠3/1                │
│      局: ⚔8900  ⛨2340 ☠18/9               │
└─────────────────────────────────────────────┘
```

### 5.3 字段与图标

| 图标 | 含义 | 颜色 | 显示形式 |
|---|---|---|---|
| `⚔` | Damage Dealt | 白（与数字同色）| HUD/表格统一 |
| `⛨` | Damage Taken | 灰白（稍暗，示意"被动"）| HUD/表格统一 |
| `☠` | Kills | 淡红（与击杀关联色）| HUD: `☠ 击/助` 合显；表格: 独立列 |
| `🤝` | Assists（同 victim 已分类伤害贡献者，不含致死一击）| 淡黄（"协助"暖色）| **仅表格**有独立列；HUD 与 ☠ 合显 |
| `♥` | Heals（v3 才加） | 淡绿（与回血关联色）| HUD/表格统一 |

**v6.6 拍板**：HUD 用 `☠ 7/3` 紧凑形式（节省横向空间）；K 键表格用独立的 ☠ + 🤝 两列（信息精确不歧义）。

### 5.4 布局细则

- **第一行（血量行）**：和 v5.3.3 完全一致（不动）
- **第二行（关卡累计）**：标识"关:" 或不加前缀，纯图标+数字
- **第三行（本局累计）**：标识"局:"
- **数字格式**：≥10000 缩写成 `12.3k`；≥1000000 缩写成 `1.2M`
- **字号**：比第一行名字略小（~80%），避免视觉压迫
- **颜色**：
  - 自己那行沿用 v5.3.3 的金色名字规则（`TEXT_GOLD`）
  - 队友保持白色
  - 离线队友整行灰白（见 §8.3）
- **不显示 stage 编号/名称**（用户明确决定）：战斗中只看数字；具体哪一关交给表格面板

### 5.5 休息室时的显示

- 休息室期间"关:" 行显示**上一关**的数据（不是 0）
- 进入下一关瞬间"关:"行清零开始累计
- 表格面板里的当前关 highlight 也对应同一逻辑

### 5.6 大厅 / 小游戏时

- 整个扩展行**隐身**（不显示"关:"/"局:" 两行），队友面板回到 v5.3.3 原状

## 6. 外观设计 B：表格面板（K 键弹出）

### 6.1 设计目标

复盘 / 分析 / 炫耀用。非战斗时打开对照每关的表现，战斗中临时开一下也能看。

### 6.2 弹出形式

- **触发键**：`K`（可配置）
- **切换逻辑**：toggle，再按 `K` 关闭
- **遮罩行为**：**Tab 式**——半透明灰色覆盖层，**游戏不暂停**，玩家仍能移动 / 攻击 / 被打
- **关闭方式**：再次按 `K` / 按 `Esc` / 按 `i` 开背包自动关

### 6.3 顶部栏（所有标签页共享）

```
┌──────────────────────────────────────────────────────────────┐
│ [当前关卡标识]  Tier 3 · Floor 27 · Boss 12                  │
│ [总表]  [分关表]                         Session: 01:23:45   │
└──────────────────────────────────────────────────────────────┘
```

- 左上显示当前所在关卡（休息室时显示"大厅"或"休息室 · 上一关: Boss12"）
- 右上计时器 = 本局 Session 总时长（从 GameID 变化开始累计）
- 两个可点击的标签页：**[总表]** 与 **[分关表]**，默认落在 **[总表]**

### 6.4 标签页 1：总表（Total Table）

**用途**：按玩家排名，每人一行，代表整局累计。

**列结构（v6.6 起八列 + 头像）**：

| 列 | 标题 | 含义 | 默认排序 |
|---|---|---|---|
| 1 | `[头像]` | 玩家皮肤头 8×8 | —— |
| 2 | `玩家` | 玩家名（自己金色；离线灰色带 `[离线]`）| —— |
| 3 | `⚔` | 本局总造伤 | **默认降序（主排序键）** |
| 4 | `⛨` | 本局总承伤 | —— |
| 5 | `☠` | 本局总击杀（含 Boss）| —— |
| 6 | `☠B` | 本局总 **Boss 击杀**（v6.6 拍板：单列；色 `Formatting.LIGHT_PURPLE`）| —— |
| 7 | `🤝` | 本局总助攻 (v6.6 新增) | —— |
| 8 | `⏱` | 本局在线时长（分:秒）| —— |
| 9（v3） | `♥` | 本局治疗队友总量 | —— |

**表底全队总计行**（v6.6 拍板）：
- 表格末尾追加一行高亮 `[全队]`，列出 `sum(⚔/⛨/☠/☠B/🤝)` + 平均 `⏱`
- 用途：控诉 + 看团队整局承担多少
- 颜色：浅金 `#FFCC66`，与玩家行视觉分开

**排序**：
- **默认**：`⚔ 降序`（"谁打得最多"）
- 点击任意列头切换该列排序（升 ⇄ 降）
- 列头显示排序箭头 `▲ / ▼`

**高亮**：
- 自己所在行：背景淡金色、名字金色
- 离线玩家：整行灰白（50% 透明度）+ 名字后加 `[离线]`

### 6.5 标签页 2：分关表（Stage Table）

**用途**：复盘每一关每个玩家的表现，按关卡层数排开。

**分组**：按 Stage 分组；同一 Stage 内按造伤降序展开每个玩家。

**列结构**（v6.6 起九列）：

| 列 | 标题 | 含义 | 排序角色 |
|---|---|---|---|
| 1 | `层` | `T{Tier}F{Floor}` 或 `T3F27` | 第一排序键（升序） |
| 2 | `关卡` | **本地化具体名称**（"紫晶迷宫"/"水漫地牢"…）由 `StageNameRegistry` 按客户端语言提供；缺失时 fallback 成 `Boss12` 类型+编号 | —— |
| 3 | `[头像]` | 玩家头像 | —— |
| 4 | `玩家` | 玩家名 | —— |
| 5 | `⚔` | 单关造伤 | 第二排序键（同关内降序） |
| 6 | `⛨` | 单关承伤 | —— |
| 7 | `☠` | 单关击杀 | —— |
| 8 | `🤝` | 单关助攻 (v6.6 新增) | —— |
| 9 | `⏱` | 单关用时 | —— |

**v6.6 关卡列细则**：
- 同关多人时，"层"和"关卡"两列只在该组**第一行**渲染（合并视觉）
- 关卡名 > 12 字符自动截断成 `紫晶迷宫…`，鼠标悬浮 tooltip 显示完整名
- 数据源：`StageNameRegistry.localizedName(kind, stageNum)`（已就绪 · 120/125 命中 · zh_cn / en_us 双语）
- **缺失语言时跨语言 fallback**（v6.6 拍板）：客户端语言为 zh 但只有 en 名 → 用 en；客户端语言为 en 但只有 zh 名 → 用 zh；两个都没有再 fallback 到类型+编号 `Boss12`

**默认排序**（**用户已确认**）：
- 主键：关卡层数**升序**（Tier×1000+Floor，从 1 → N）
- 次键：同一关内按 `⚔ 降序`（每关谁最强）

**当前关卡高亮**（v6.6 拍板）：
- 正在进行的关卡整组**描金边**（或金色背景淡化），关卡名后缀 `⭐ (进行中)`
- 休息室期间：高亮"上一关"整组（金边），关卡名后缀 `(休息中)`，但 `⏱` 列冻结在该关结束时刻
- 大厅 / 小游戏：不高亮任何组

### 6.6 滚动与溢出

- 总表 / 分关表行数超过可视区 → 右侧出现 GUI 风格的滚动条
- 鼠标滚轮滚动
- 滚动条支持拖拽
- 当前关卡组永远在视野内（自动滚动 keep-alive）

### 6.7 空状态

- 没任何数据时（Session 刚开始、首次进入）
  ```
  ╭─────────────────────────────────╮
  │       尚无数据                  │
  │  进入第一关后开始统计           │
  ╰─────────────────────────────────╯
  ```

### 6.8 颜色 / 风格

- 整体走 vanilla **Tab 菜单** 风 (`#181818` ~55% 透明 背景，白字)
- 自己 金色 `#FFAA00`

## 8. 特殊状态处理

### 8.1 癫狂 / 失明 / Boss 栏失联

**不中断采集**。服务端依然在记录 `Damage` scoreboard 和四层血 delta，客户端 HUD 即便暂时看不到血条，表格面板打开后数据依然正确。

### 8.2 复活 (`tag=Respawning`)

- 死亡瞬间→ `kills_mob` 不算此次死亡（仅算 death of player，不纳入玩家数据）
- 复活等待期间：玩家承伤 / 造伤都是 0（地图不让他动），天然为 0
- 复活完成 → 正常累计

### 8.3 离线玩家

- **服务端**：玩家离开时冻结其数据（不清理），`last_seen_tick` 打上时间戳
- **客户端显示**：
  - 嵌入 HUD：该玩家行消失（因为队友 Boss 栏已不含他）
  - 表格面板：继续显示，整行灰 50%，名字后追加 `[离线]`
- 玩家回来：继续从冻结点累加；`[离线]` 标签消失

### 8.4 Session 切换（玩家开新局）

- `GameID` 跳变 → 旧 Session 数据归档到 `ctt_stats.dat` 的 `history[]`（可选将来做历史对局查看）
- 新 Session 从零开始
- 表格面板弹出时只显示当前 Session；历史对局 v1 不在 UI 暴露

### 8.5 掉存档 / 崩溃

- 每 30 秒或每关结束时 flush 一次 `ctt_stats.dat`
- 重启后最多丢 30 秒

### 8.6 测试假人（TestDummy）

- 休息室期间不采集 → 假人伤害**自动被过滤**
- 不做白名单；也就是说就算有人在非休息室造出假人也会被记（罕见场景，保守处理）

## 10. 分发形态

- **单 jar**：`cake-tower-hud-plus-6.x.x.jar`
- **环境**：`environment: "*"`（客户端 + 服务端同包）
- **入口**：
  - `client` → HUD（v5 所有 + v6 新 UI）
  - `main` → 统计采集 / 持久化 / payload 同步
- **部署建议**：
  - **单机本地存档**：丢到 `mods/`，完美工作（integrated server 两边都加载）
  - **局域网联机（LAN open to）**：房主装 mod 够用；队友装不装都能正常玩，装了能看到 HUD
  - **专用服务器**：服务器和每个玩家都装，才能看到其他玩家的采集数据；只服务器装 → 数据正确记录但没人能看；只玩家装 → 看不到数据（没人在服务端采集）

## 12. 非目标（明确不做）

- ❌ **全地图战绩排行榜 / 云端同步**：纯本地存档
- ❌ **分职业 / 分武器 类型细分**：只看总量，不拆类型
- ❌ **DPS 实时曲线图**：v1 不做，只做累计
- ❌ **分享 / 截图上传功能**：交给玩家自己 F2
- ❌ **历史对局浏览器**：v1 / v6.6 只看当前 Session（结束的 Session 仍持久化但不进 UI）
- ❌ **友军伤害拒绝统计**：地图允许友伤存在，v1 不特殊剔除
